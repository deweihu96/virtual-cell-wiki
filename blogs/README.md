# blogs

Practical, hands-on writeups about virtual cell models and related work — competition
logs, submission postmortems, opinion pieces, tutorials. One Markdown file per post.
These are **not peer-reviewed**: they're experience and opinion. Keep this content
mentally separate from `wiki/`, which holds structured references to published models.

The Arc **Virtual Cell Challenge** is one common topic here, but not the only one —
anything hands-on about virtual cell modeling belongs.

Frontmatter (all fields optional except title/source):

```yaml
---
title: "..."
author: "..."
source: <url>
date: YYYY-MM-DD
topic: <e.g. virtual-cell-challenge, or free text>
tags: [practical, ...]
---
```

Body is free-form Markdown. No fixed schema, no `not stated` convention — summarize
whatever the post actually offers.
